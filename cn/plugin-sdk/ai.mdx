本页不新增实时 `/mcp` 后端接口。它定义后续如何通过 Mintlify 或自定义只读 MCP 服务安全暴露 `cn/plugin-sdk/` 文档。

## 目标

让 AI 助手在编写 autClaw 插件时只检索需要的文档。

推荐流程：

```text
user question
  ↓
search_autclaw_plugin_docs(query, limit)
  ↓
read_autclaw_plugin_doc(path, start_line, max_lines)
  ↓
optionally read schema/type/example files
```

## 推荐 MCP 工具

### search_autclaw_plugin_docs

输入：

```json
{
  "query": "replyImage parameters",
  "limit": 5
}
```

输出：

```json
{
  "results": [
    {
      "title": "middleware.js Reference",
      "path": "cn/plugin-sdk/javascript.mdx",
      "section": "Media replies",
      "snippet": "replyImage(source, options?) returns SendReceipt"
    }
  ]
}
```

规则：

- 默认 `limit=5`；最大 `10`。
- 只返回标题、路径、章节和短摘要。
- 搜索结果不要返回完整文档。

### read_autclaw_plugin_doc

输入：

```json
{
  "path": "cn/plugin-sdk/javascript.mdx",
  "start_line": 120,
  "max_lines": 120
}
```

规则：

- 只允许这些路径：
  - `cn/plugin-sdk/**`
  - `cn/plugin-sdk/media.mdx`
- 默认最多读取 `120` 行。
- 硬上限为 `240` 行。
- 大文件必须按页或行范围读取。
- 不要读取插件源码、数据库、日志、环境文件、密钥或用户数据。

### list_autclaw_plugin_docs

输入：

```json
{
  "path": "cn/plugin-sdk"
}
```

从 `ai-index.json` 返回文件名和简短说明。

### get_autclaw_plugin_schema

输入：

```json
{
  "name": "plugin-manifest"
}
```

允许的名称：

- `plugin-manifest` → `cn/plugin-sdk/schemas/plugin-manifest.schema.json`
- `middleware-runtime` → `cn/plugin-sdk/schemas/middleware-runtime.schema.json`
- `middleware-js-types` → `cn/plugin-sdk/types/middleware-js.d.ts`

## 静态来源

| 来源 | 用途 |
|---|---|
| `llms.txt` | 小型 LLM 入口。 |
| `cn/plugin-sdk/ai-index.json` | 搜索和列表种子。 |
| `cn/plugin-sdk/javascript.mdx` | 主要 JavaScript SDK 参考。 |
| `cn/plugin-sdk/python.mdx` | 主要 Python SDK 参考。 |
| `cn/plugin-sdk/manifest.mdx` | Header / manifest 契约。 |
| `cn/plugin-sdk/types/middleware-js.d.ts` | 精确 JavaScript 签名。 |
| `cn/plugin-sdk/schemas/*.schema.json` | 机器可读契约。 |
| `cn/plugin-sdk/media.mdx` | 媒体行为细节。 |

## 安全边界

文档 MCP 必须是只读的。

不要暴露：

- 任意 shell 命令。
- 运行时数据库 bucket。
- 文档允许列表之外的用户插件源码。
- `plugin/.tmpfs`、`.env`、日志、token、配置文件、私钥、cookie 或数据库文件。
- 保存插件、安装依赖、重启、更新或删除等写操作。

## 上下文预算规则

- 先搜索，再读取。
- 只读取能回答问题的最小章节。
- 精确字段名优先查 schema / 类型。
- 用法示例和注意事项优先查说明文档。
- 避免 `get_all_docs` 这类工具。

## 后续实现说明

如果后续把它实现为 Go 接口，路由变更必须遵循 `docs/chi-best-practices.md`，补充测试并运行：

```bash

```

然后更新 `docs/routes.md`。
