---
title: "JavaScript middleware reference"
description: "在 JavaScript 插件中读取上下文、回复消息、发送媒体、保存数据、处理路由和定时任务。"
---

`plugin/scripts/middleware.js` 是 JavaScript 普通插件的 SDK。你通过它创建 `Sender`、读取消息上下文、回复文本或媒体、保存数据、下载文件、等待用户输入和管理定时任务。

<Note>
  新插件优先使用 `const { Sender, getSenderID } = require('./middleware.js')`。不要在插件里硬编码凭据、端口或服务地址。
</Note>

## 快速开始

```js
//[author: your-name]
//[runtime: node@22.22.1]
//[title: Echo]
//[rule: echo (.+)]
const { Sender, getSenderID } = require('./middleware.js')

async function main() {
  const sender = new Sender(getSenderID())
  const text = await sender.param(1)
  await sender.reply(text || '请输入内容')
}

main().catch((error) => console.error(error))
```

## 导入方式

```js
const {
  Sender,
  Cron,
  getSenderID,
  bucketGet,
  bucketSet,
  fileDownload,
  downloadAdapterFile,
  importModule,
} = require('./middleware.js')

const sender = new Sender(getSenderID())
```

常用规则：

- 使用 `Sender` 处理当前消息、当前路由请求或当前事件。
- 使用顶层函数做全局操作，例如 `bucketGet(...)`、`pushImage(...)`、`fileDownload(...)`。
- 大多数方法返回运行结果；失败会抛出 `Error`，建议用 `try/catch` 处理。

## 常用返回值

### SendReceipt

文本、媒体和撤回类方法通常返回发送回执。

```json
{
  "ok": true,
  "message_id": "123456",
  "messageid": "123456",
  "message_ids": ["123456"],
  "raw": {},
  "action": "sendText"
}
```

适配器可能返回额外字段。判断失败时优先检查 `receipt.ok === false`，消息 ID 同时兼容 `message_id` 和 `messageid`。

### FileDownloadResult

`fileDownload(...)` 和 `downloadAdapterFile(...)` 返回本地文件信息。

```json
{
  "path": "/opt/autclaw/file/demo.png",
  "file_path": "/opt/autclaw/file/demo.png",
  "file_name": "demo.png",
  "mime_type": "image/png",
  "file_size": 12345
}
```

### MediaItem

`sender.getMediaItems()` 返回当前事件或最近一次 `listen/input` 捕获到的媒体项。

```ts
type MediaItem = {
  type: 'text' | 'markdown' | 'image' | 'voice' | 'video' | 'file' | string
  text?: string
  source?: string
  url?: string
  path?: string
  file?: string
  name?: string
  file_name?: string
  filename?: string
  mime_type?: string
  mimeType?: string
}
```

把用户上传的文件项传给 `downloadAdapterFile(...)`，不要手动解析平台原始字段。

## Sender

创建一个 `Sender`：

```js
const sender = new Sender(getSenderID())
```

### 读取消息上下文

| 方法 | 返回 | 说明 |
|---|---|---|
| `getImtype()` | `Promise<string>` | 当前 IM 类型。 |
| `getUserID()` | `Promise<string>` | 当前用户 ID。 |
| `getUserName()` | `Promise<string>` | 当前用户昵称。 |
| `getUserAvatarUrl()` | `Promise<string>` | 当前用户头像 URL。 |
| `getChatID()` | `Promise<string>` | 当前会话 ID。 |
| `getChatName()` | `Promise<string>` | 当前会话名称。 |
| `getGroupName()` | `Promise<string>` | 群名，等同于会话名称。 |
| `isAdmin()` | `Promise<boolean>` | 当前用户是否为管理员。 |
| `isAI()` | `Promise<boolean>` | 当前消息是否来自 AI 触发上下文。 |
| `getAIParam()` | `Promise<string>` | AI 参数。 |
| `getMessage()` | `Promise<string>` | 当前消息文本。 |
| `getMessageID()` | `Promise<string>` | 当前消息 ID。 |
| `getHistoryMessageIDs(number)` | `Promise<string[]>` | 最近消息 ID。 |
| `param(index)` | `Promise<string>` | 读取 `rule` 正则捕获，`index` 从 `1` 开始。 |

```js
const user = await sender.getUserName()
const message = await sender.getMessage()
await sender.reply(`${user} 发送了：${message}`)
```

### 回复文本和媒体

```js
await sender.reply('普通文本')
await sender.replyMarkdown('# 标题')
await sender.replyImage('https://example.com/a.png')
await sender.replyVoice('./voice.wav')
await sender.replyVideo('./demo.mp4')
await sender.replyFile('./report.csv', { filename: 'report.csv' })
await sender.replyMixed([
  { type: 'text', text: '结果：' },
  { type: 'image', source: './result.png' },
])
```

| 方法 | 参数 | 返回 |
|---|---|---|
| `reply(text)` | `string` | `Promise<SendReceipt>` |
| `replyMarkdown(markdown)` | `string` | `Promise<SendReceipt>` |
| `replyImage(source, options?)` | 媒体来源 | `Promise<SendReceipt>` |
| `replyVoice(source, options?)` | 媒体来源 | `Promise<SendReceipt>` |
| `replyVideo(source, options?)` | 媒体来源 | `Promise<SendReceipt>` |
| `replyFile(source, options?)` | 媒体来源 | `Promise<SendReceipt>` |
| `replyMixed(items, options?)` | mixed 数组 | `Promise<SendReceipt>` |
| `returnValue(text)` | `string` | 运行结果 |

媒体来源和 mixed 写法见 [`Media sending`](/cn/plugin-sdk/media)。

### 控制流程

| 方法 | 参数 | 说明 |
|---|---|---|
| `setContinue()` | 无 | 允许继续匹配后续插件。 |
| `recallMessage(messageid)` | 消息 ID | 撤回消息。 |
| `breakIn(content)` | 文本 | 把新文本重新送入消息处理。 |

```js
if (await sender.isAdmin()) {
  await sender.setContinue()
}
```

### 等待输入和支付

| 方法 | 参数 | 返回 |
|---|---|---|
| `listen(waitTime, recallTime?, scope?)` | 等待毫秒数，可选撤回毫秒数，可选 `group` | 用户输入、事件对象或超时标记。 |
| `input(waitTime, recallTime?, scope?)` | 同 `listen` | 用户输入、事件对象或超时标记。 |
| `waitPay(exitcode, timeout, amount?)` | 订单标识、超时毫秒数、可选金额 | `timeout` / `busy` / 支付成功对象。 |
| `atWaitPay()` | 无 | 是否处于等待支付状态。 |

```js
await sender.reply('请在 60 秒内发送验证码')
const code = await sender.input(60000)
await sender.reply(`收到：${typeof code === 'string' ? code : JSON.stringify(code)}`)
```

如果允许同群任意成员回答，把第三个参数设为 `'group'`：

```js
const answer = await sender.input(60000, 0, 'group')
```

`listen(waitTime, 'group')` 也会把第二个字符串参数识别为作用域。

### 事件和媒体项

| 方法 | 返回 | 说明 |
|---|---|---|
| `getPluginName()` | `Promise<string>` | 当前插件名称。 |
| `getPluginVersion()` | `Promise<string>` | 当前插件版本。 |
| `getEventType()` | `Promise<string>` | 当前事件类型。 |
| `setEventType(eventType)` | `Promise<boolean>` | 设置事件类型。 |
| `getEventData()` | `Promise<any>` | 当前事件数据。 |
| `setEventData(eventData)` | `Promise<boolean>` | 设置事件数据。 |
| `getMediaItems()` | `Promise<MediaItem[]>` | 当前事件或最近一次等待输入捕获到的媒体。 |

```js
const items = await sender.getMediaItems()
const fileItem = items.find((item) => item.type === 'file')
```

### HTTP 路由请求

在 manifest 中声明 `router` 和 `method` 后，可以用这些方法读取请求并响应。

```js
//[router: /demo/{id}]
//[method: post]
const { Sender, getSenderID } = require('./middleware.js')

const sender = new Sender(getSenderID())
const params = await sender.getRouterParams()
const body = await sender.getRouterBody()
await sender.response({ ok: true, id: params.id, body })
```

| 方法 | 返回 | 说明 |
|---|---|---|
| `getRouterPath()` / `getRouter()` | `Promise<string>` | 当前路由路径。 |
| `getRouterParams()` | `Promise<Record<string, string>>` | 路由参数。 |
| `getRouterMethod()` / `getMethod()` | `Promise<string>` | 请求方法。 |
| `getRouterHeaders()` | `Promise<object>` | 请求头。 |
| `getRouterCookies()` | `Promise<object>` | Cookie。 |
| `getRouterBody()` | `Promise<any>` | 请求体。 |
| `getRouterData()` | `Promise<string>` | 请求体 JSON 字符串。 |
| `response(data)` | `Promise<boolean>` | 返回 JSON 数据。 |

## 存储

### 默认存储

默认存储适合保存少量插件状态。

```js
await set('last_user', '10001')
const user = await get('last_user')
await del('last_user')
```

| 函数 | 参数 | 返回 |
|---|---|---|
| `get(key)` | `string` | `Promise<string>`，不存在时返回空字符串。 |
| `set(key, value)` | key + 字符串值 | `Promise<boolean>` |
| `del(key)` | `string` | `Promise<boolean>` |

<Warning>
  当前存储会把 `value` 按字符串保存。需要保存对象时，请先 `JSON.stringify(...)`，读取后再 `JSON.parse(...)`。
</Warning>

### 命名 bucket

```js
await bucketSet('my_plugin', 'config', JSON.stringify({ enabled: true }))
const raw = await bucketGet('my_plugin', 'config')
const keys = await bucketAllKeys('my_plugin')
const all = await bucketAll('my_plugin')
```

| 函数 / Sender 方法 | 参数 | 返回 |
|---|---|---|
| `bucketGet(bucket, key)` | bucket + key | `Promise<string>`，不存在时返回空字符串。 |
| `bucketSet(bucket, key, value)` | bucket + key + 字符串值 | `Promise<boolean>` |
| `bucketDel(bucket, key)` | bucket + key | `Promise<boolean>` |
| `bucketKeys(bucket?, value?)` | bucket 和可选值过滤 | `Promise<string[]>` |
| `bucketAllKeys(bucket)` | bucket | `Promise<string[]>` |
| `bucketAll(bucket)` | bucket | `Promise<Record<string, string>>` |

`Sender` 也提供同名 bucket 方法，适合想把调用绑定到当前会话上下文的场景。

## 文件和媒体工具

| 函数 | 参数 | 返回 |
|---|---|---|
| `fileDownload(url, path?)` | 下载远程 URL、`base64://...` 或 data URI | `Promise<FileDownloadResult>` |
| `downloadAdapterFile(file, options?, timeout?)` | 下载用户上传的文件项 | `Promise<FileDownloadResult>` |
| `getGroupMemberList(groupIDOrOptions?, options?, timeout?)` | 获取群成员列表 | 群成员列表对象 |

```js
const downloaded = await fileDownload('https://example.com/a.png', 'images')
await sender.replyImage(downloaded.path)

const items = await sender.getMediaItems()
const fileItem = items.find((item) => item.type === 'file')
if (fileItem) {
  const saved = await downloadAdapterFile(fileItem, { path: 'imports' })
  await sender.reply(`已保存：${saved.file_name}`)
}
```

更多媒体说明见 [`Media sending`](/cn/plugin-sdk/media)。

## 主动推送

主动推送不依赖当前 `Sender`，需要指定 IM 类型、群和用户。

| 函数 | 参数 | 说明 |
|---|---|---|
| `push(imType, groupCode, userID, title, content)` | 目标 + 标题 + 文本 | 推送文本。当前 helper 不返回结果。 |
| `pushImage(imType, groupCode, userID, title, imageUrl)` | 目标 + 图片 | 推送图片。 |
| `pushVoice(imType, groupCode, userID, title, voiceUrl)` | 目标 + 语音 | 推送语音。 |
| `pushVideo(imType, groupCode, userID, title, videoUrl)` | 目标 + 视频 | 推送视频。 |
| `pushFile(imType, groupCode, userID, title, fileUrl, options?)` | 目标 + 文件 | 推送文件。 |
| `pushMixed(imType, groupCode, userID, title, items)` | 目标 + mixed items | 推送混合消息。 |

```js
await pushImage('qq', '123456', '', '日报', './daily.png')
```

## 依赖和工具函数

### importModule

```js
const dayjs = await importModule('dayjs')

const axios = await importModule('axios', { package: 'axios', version: '1.7.9', manager: 'npm' })
```

`importModule(module, versionOrOptions?, manager?)` 会先确保依赖可用，再返回 `require(module)` 的结果。包名与导入名不一致时，使用 `{ package: '真实包名' }`。

## Cron

`Cron` 用来管理运行中的定时指令。它不同于 manifest 里的 `cron`：manifest `cron` 用来让当前插件自触发；`Cron` 类用来增删改查运行时定时指令。

```js
const cron = new Cron()
const items = await cron.getCrons()
const item = await cron.getCron(1)
cron.addCron('0 9 * * *', false, 'daily reminder', true, null, '每日提醒', '', '', '')
cron.updateCron(1, '0 10 * * *', false, 'daily reminder', true, null, '每日提醒', '', '', '')
cron.delCron(1)
```

| 方法 | 参数 | 返回 |
|---|---|---|
| `getCrons()` | 无 | 定时指令列表。 |
| `getCron(id)` | ID | 单个定时指令。 |
| `addCron(cron, isOnce?, cmd?, isToSelf?, toOthers?, memo?, disguiseImtype?, disguiseGroup?, disguiseUser?)` | 定时配置 | 当前 JS helper 发起请求但不返回结果。 |
| `updateCron(id, cron, isOnce, cmd, isToSelf, toOthers, memo, disguiseImtype, disguiseGroup, disguiseUser)` | 更新配置 | 当前 JS helper 发起请求但不返回结果。 |
| `delCron(id)` | ID | 当前 JS helper 发起请求但不返回结果。 |

## 群管理

这些方法取决于当前 IM 平台支持情况；不支持时可能无效果或返回错误。

```js
await sender.groupInviteIn(friendID, groupID)
await sender.groupKick(userID)
await sender.groupBan(userID, 600)
await sender.groupUnban(userID)
await sender.groupWholeBan()
await sender.groupWholeUnban()
await sender.groupNoticeSend('公告内容')
```

## 推荐模板

```js
//[author: your-name]
//[runtime: node@22.22.1]
//[title: Safe template]
//[rule: demo (.+)]
const { Sender, getSenderID } = require('./middleware.js')

async function main() {
  const sender = new Sender(getSenderID())
  const value = await sender.param(1)
  await sender.reply(value || '请输入内容')
}

main().catch(async (error) => {
  console.error(error && error.stack ? error.stack : error)
})
```

## 常见错误

- `sender.param(1)` 从 `1` 开始，不是 `0`。
- `set(...)` / `bucketSet(...)` 保存字符串；对象请手动 JSON 序列化。
- 不要绕过 SDK 直接请求 autClaw 服务；优先使用本页方法。
- 下载用户上传文件时，使用 `sender.getMediaItems()` + `downloadAdapterFile(...)`。
- 大文件和媒体发送失败时，检查回执里的 `error` 并给用户明确提示。
